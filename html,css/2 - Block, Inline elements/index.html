Блочные элементы

Исторически HTML-элементы было принято делить на блочные и строчные. Блочные элементы занимают всю ширину своего родителя (контейнера), формально создавая «блок» (отсюда и название). Браузеры обычно отображают блочные элементы с переводом строки до и после элемента. Блочные элементы можно представить в виде стопки коробок. Следующий пример демонстрирует, как это выглядит:

Как правило, блочные элементы могут содержать строчные элементы и другие блочные элементы. Неотъемлемой частью этого структурного различия является идея, что блочные элементы создают «более крупную» структуру, чем встроенные элементы.

По умолчанию блочные элементы начинаются с новой строки, а строчные могут начинаться в любом месте строки.

header - блочный элемент
main - блочный элемент
footer - блочный элемент
div - блочный элемент

p - Этот абзац — блочный элемент

h1-h6 - блочные элементы

    ul - это блочный элемент
  • li (List item) - блочный элемент
    ol - это блочный элемент
  1. li (List item) - блочный элемент
nav - блочный элемент

Строчные элементы

Строчные элементы это те, которые занимают только то пространство, которое ограничено тегами, определяющими строчный элемент (необходимое для отображения их содержимого) и не нарушая потока содержимого (не требующее новой строки после каждого элемента) Так же в строчные элементы нельзя вкладывать блочные элементы

По умолчанию встроенные элементы не заставляют новую строку начинаться в потоке документа. С другой стороны, элементы блока обычно вызывают разрыв строки (хотя, как обычно, это можно изменить с помощью CSS).

span - строчный элемент b - строчный элемент i - строчный элемент small - строчный элемент strong - строчный элемент a - строчный элемент img - строчный элемент input - строчный элемент
